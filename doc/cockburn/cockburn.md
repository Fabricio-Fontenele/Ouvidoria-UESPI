# Cockburn dos requisitos

Documento de casos de uso derivado de `doc/basic/requisitos.md`.
Este documento deve herdar o escopo, as regras e as restricoes do documento basico de requisitos.

## Caso de uso de cadastro de usuario

| Campo                         | Descricao                                                                                                                                                                                                                                                                                          |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                        | UC-01                                                                                                                                                                                                                                                                                              |
| **Nome**                      | Cadastrar usuario                                                                                                                                                                                                                                                                                  |
| **Objetivo**                  | Permitir o cadastro de um novo usuario no sistema                                                                                                                                                                                                                                                  |
| **Ator primario**             | Usuario                                                                                                                                                                                                                                                                                            |
| **Interessados e interesses** | **Usuario:** deseja criar conta para acessar o sistema. <br> **Ouvidoria:** deseja permitir acesso apenas a usuarios cadastrados.                                                                                                                                                                  |
| **Gatilho**                   | O usuario seleciona a opcao de cadastro                                                                                                                                                                                                                                                            |
| **Pre-condicoes**             | O usuario ainda nao possui cadastro ativo com o e-mail informado.                                                                                                                                                                                                                                  |
| **Pos-condicoes**             | O usuario e cadastrado com sucesso e passa a poder se autenticar no sistema.                                                                                                                                                                                                                       |
| **Garantias minimas**         | O sistema nao conclui cadastro com dados obrigatorios ausentes ou invalidos.                                                                                                                                                                                                                       |
| **Garantias de sucesso**      | A conta e criada com e-mail unico e dados validados.                                                                                                                                                                                                                                               |
| **Fluxo principal**           | 1. O usuario acessa a funcionalidade de cadastro. <br> 2. O sistema solicita os dados obrigatorios. <br> 3. O usuario informa os dados cadastrais. <br> 4. O sistema valida as informacoes fornecidas. <br> 5. O sistema registra o novo usuario. <br> 6. O sistema confirma o cadastro realizado. |
| **Extensoes**                 | 4.1 E-mail ja cadastrado: o sistema informa a duplicidade e impede o cadastro. <br> 4.2 Dados invalidos: o sistema informa os erros e solicita correcao.                                                                                                                                           |
| **Regras de negocio**         | RN-01: O e-mail do usuario deve ser unico. <br> RN-02: O cadastro deve respeitar os perfis de acesso previstos.                                                                                                                                                                                    |
| **Requisitos especiais**      | O cadastro deve ser acessivel em desktop e dispositivos moveis.                                                                                                                                                                                                                                    |
| **Requisitos relacionados**   | RF01, RF05, RNF04, RNF05, RNF07                                                                                                                                                                                                                                                                    |

| **Nome**                      | Autenticar usuario                                                                                                                                                                                                                                                               |

| **Ator primario**             | Usuario                                                                                                                                                                                                                                                                          |
| **Interessados e interesses** | **Usuario:** deseja acessar sua conta. <br> **Ouvidoria:** deseja restringir o acesso a perfis autorizados.                                                                                                                                                                      |
| **Gatilho**                   | O usuario seleciona a opcao de login                                                                                                                                                                                                                                             |
| **Pre-condicoes**             | O usuario possui cadastro ativo no sistema.                                                                                                                                                                                                                                      |
| **Pos-condicoes**             | O usuario inicia sessao e recebe acesso conforme perfil autorizado.                                                                                                                                                                                                              |
| **Garantias minimas**         | Credenciais invalidas nao concedem acesso ao sistema.                                                                                                                                                                                                                            |
| **Garantias de sucesso**      | O sistema autentica o usuario e aplica as permissoes do perfil.                                                                                                                                                                                                                  |

| **Garantias minimas**         | Nenhuma senha e alterada sem validacao do fluxo seguro de recuperacao.                                                                                                                                                                                                                                                                                     |

O sistema envia o mecanismo de recuperacao por canal seguro. <br> 5. O usuario informa a nova senha conforme o fluxo recebido. <br> 6. O sistema valida a solicitacao e redefine a senha. |

| **Requisitos especiais**      | O fluxo deve ser compativel com desktop e dispositivos moveis.                                                                                                                                                                                                                                                                                             |
| **Requisitos relacionados**   | RF03, RNF07                                                                                                                                                                                                                                                                                                                                                |

| **Gatilho**                   | O usuario inicia o registro de uma manifestacao                                                                                                                                                                                                                                                                                                                                                                       |

| **Extensoes**                 | 3.1 Dados obrigatorios ausentes: o sistema informa os campos pendentes. <br> 4.1 Anexo invalido: o sistema rejeita o arquivo conforme regras de seguranca. <br> 5.1 Manifestacao anonima ou sigilosa: o sistema aplica a politica institucional correspondente.                                                                                                                                                       |
| **Regras de negocio**         | RN-01: Toda manifestacao deve possuir protocolo unico. <br> RN-02: Toda manifestacao deve estar vinculada a campus e unidade administrativa. <br> RN-03: Os tipos permitidos sao denuncia, reclamacao, sugestao e elogio.                                                                                                                                                                                             |
| **Requisitos especiais**      | O formulario deve ser responsivo, acessivel e registrar a manifestacao em tempo aceitavel.                                                                                                                                                                                                                                                                                                                            |
| **Requisitos relacionados**   | RF06, RF07, RF08, RF09, RF10, RF11, RN01, RN02, RN03, RNF04, RNF05, RNF11                                                                                                                                                                                                                                                                                                                                             |

| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                        | UC-05                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **Nome**                      | Acompanhar manifestacao                                                                                                                                                                                                                                                                                                                                                                                                |
| **Objetivo**                  | Permitir consulta do andamento da manifestacao e interacao por mensagens entre participantes autorizados                                                                                                                                                                                                                                                                                                               |
| **Ator primario**             | Manifestante                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Interessados e interesses** | **Manifestante:** deseja acompanhar status, historico e respostas da ouvidoria. <br> **Ouvidoria:** deseja manter a comunicacao rastreavel durante o tratamento.                                                                                                                                                                                                                                                       |
| **Gatilho**                   | O manifestante acessa uma manifestacao registrada                                                                                                                                                                                                                                                                                                                                                                      |
| **Pre-condicoes**             | Existe manifestacao registrada e o ator possui permissao de acesso.                                                                                                                                                                                                                                                                                                                                                    |
| **Pos-condicoes**             | O manifestante consulta o historico e, quando permitido, registra nova interacao no chamado.                                                                                                                                                                                                                                                                                                                           |
| **Garantias minimas**         | O sistema nao exibe manifestacoes para usuarios sem autorizacao.                                                                                                                                                                                                                                                                                                                                                       |
| **Garantias de sucesso**      | O historico, o status atual e as interacoes ficam disponiveis de forma rastreavel.                                                                                                                                                                                                                                                                                                                                     |
| **Fluxo principal**           | 1. O manifestante acessa suas manifestacoes. <br> 2. O sistema lista as manifestacoes disponiveis conforme autorizacao. <br> 3. O manifestante seleciona uma manifestacao. <br> 4. O sistema exibe status atual, historico de movimentacoes e interacoes. <br> 5. O manifestante envia mensagem complementar, quando a manifestacao estiver aberta para interacao. <br> 6. O sistema registra a mensagem no historico. |
| **Extensoes**                 | 2.1 Nenhuma manifestacao disponivel: o sistema informa a ausencia de registros acessiveis. <br> 5.1 Manifestacao sem permissao de interacao: o sistema bloqueia o envio de mensagem.                                                                                                                                                                                                                                   |
| **Regras de negocio**         | RN-01: O acompanhamento deve respeitar a autorizacao de acesso. <br> RN-02: O tratamento deve manter historico de mensagens e alteracoes de status.                                                                                                                                                                                                                                                                    |
| **Requisitos especiais**      | O sistema deve notificar o usuario sobre atualizacoes relevantes da manifestacao, respeitando as restricoes de casos anonimos.                                                                                                                                                                                                                                                                                         |
| **Requisitos relacionados**   | RF12, RF13, RF14, RF17, RN05, RNF09                                                                                                                                                                                                                                                                                                                                                                                    |

| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                        | UC-06                                                                                                                                                                                                                                                                                                                            |
| **Nome**                      | Encerrar e avaliar manifestacao                                                                                                                                                                                                                                                                                                  |
| **Objetivo**                  | Permitir que o manifestante encerre a manifestacao e registre a avaliacao do atendimento                                                                                                                                                                                                                                         |
| **Ator primario**             | Manifestante                                                                                                                                                                                                                                                                                                                     |
| **Interessados e interesses** | **Manifestante:** deseja concluir formalmente o atendimento. <br> **Ouvidoria:** deseja registrar o fechamento e obter avaliacao do atendimento.                                                                                                                                                                                 |
| **Gatilho**                   | O manifestante decide concluir uma manifestacao com tratamento encerravel                                                                                                                                                                                                                                                        |
| **Pre-condicoes**             | A manifestacao atende as condicoes de fechamento definidas pela ouvidoria.                                                                                                                                                                                                                                                       |
| **Pos-condicoes**             | A manifestacao e encerrada e a avaliacao do atendimento pode ser registrada.                                                                                                                                                                                                                                                     |
| **Garantias minimas**         | O sistema nao encerra manifestacoes fora das condicoes permitidas.                                                                                                                                                                                                                                                               |
| **Garantias de sucesso**      | O encerramento fica registrado no historico e a avaliacao e associada a manifestacao finalizada.                                                                                                                                                                                                                                 |
| **Fluxo principal**           | 1. O manifestante acessa a manifestacao. <br> 2. O sistema informa que a manifestacao atende as condicoes de encerramento. <br> 3. O manifestante confirma o encerramento. <br> 4. O sistema registra o encerramento. <br> 5. O sistema disponibiliza a avaliacao do atendimento. <br> 6. O manifestante registra sua avaliacao. |
| **Extensoes**                 | 2.1 Manifestacao ainda nao encerravel: o sistema informa a restricao. <br> 6.1 Manifestante opta por nao avaliar: o sistema conclui apenas o encerramento.                                                                                                                                                                       |
| **Regras de negocio**         | RN-01: Manifestacoes finalizadas nao aceitam edicao direta do conteudo original. <br> RN-02: A avaliacao deve ficar vinculada a manifestacoes finalizadas.                                                                                                                                                                       |
| **Requisitos especiais**      | O encerramento e a avaliacao devem permanecer rastreaveis no historico da manifestacao.                                                                                                                                                                                                                                          |
| **Requisitos relacionados**   | RF15, RF16, RN06, RNF09                                                                                                                                                                                                                                                                                                          |

| **Pre-condicoes**             | Existe interacao previa com a IA e a demanda e compativel com abertura de manifestacao.                                                                                                                                                                                                                                                                                                     |
| **Pos-condicoes**             | O formulario e apresentado com campos previamente preenchidos para revisao, edicao e confirmacao do usuario.                                                                                                                                                                                                                                                                                |
| **Garantias minimas**         | Nenhuma manifestacao e registrada sem revisao e confirmacao do usuario.                                                                                                                                                                                                                                                                                                                     |
| **Garantias de sucesso**      | O usuario aproveita ou ajusta o pre-preenchimento e segue para o registro formal da manifestacao.                                                                                                                                                                                                                                                                                           |
| **Fluxo principal**           | 1. A IA identifica necessidade de tratamento formal. <br> 2. O sistema organiza as informacoes fornecidas pelo usuario. <br> 3. O sistema apresenta o formulario da manifestacao com campos previamente preenchidos. <br> 4. O usuario revisa e edita os dados, quando necessario. <br> 5. O usuario confirma o envio. <br> 6. O sistema segue o fluxo regular de registro de manifestacao. |
| **Extensoes**                 | 3.1 Informacoes insuficientes: o sistema solicita complementacao antes de pre-preencher. <br> 4.1 Usuario rejeita o pre-preenchimento: o sistema permite preenchimento manual.                                                                                                                                                                                                              |
| **Regras de negocio**         | RN-01: A IA pode apoiar o preenchimento, mas nao substitui a confirmacao do usuario. <br> RN-02: O envio final deve respeitar as mesmas validacoes do registro padrao de manifestacao.                                                                                                                                                                                                      |
| **Requisitos especiais**      | O fluxo assistido deve ser claro, editavel e rastreavel.                                                                                                                                                                                                                                                                                                                                    |
| **Requisitos relacionados**   | RF27, RF06, RF07, RF08, RF10, RN07                                                                                                                                                                                                                                                                                                                                                          |
