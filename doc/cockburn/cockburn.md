# Cockburn dos requisitos

## Caso de uso de autenticação

| Campo                         | Descrição                                                                                                                                                                                                                                                                                            |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                        | UC-01                                                                                                                                                                                                                                                                                                |
| **Nome**                      | Cadastrar usuário                                                                                                                                                                                                                                                                                    |
| **Objetivo**                  | Permitir que um novo usuário se cadastre no sistema                                                                                                                                                                                                                                                  |
| **Ator primário**             | Usuário                                                                                                                                                                                                                                                                                              |
| **Interessados e Interesses** | **Usuário:** deseja criar conta para acessar o sistema. <br> **UESPI/Ouvidoria:** deseja permitir acesso apenas a usuários devidamente cadastrados.                                                                                                                                                  |
| **Gatilho**                   | O usuário seleciona a opção de cadastro                                                                                                                                                                                                                                                              |
| **Pré-condições**             | O usuário ainda não possui cadastro ativo no sistema. <br> O sistema está disponível.                                                                                                                                                                                                                |
| **Pós-condições**             | O usuário é cadastrado com sucesso e pode acessar o sistema. <br> O sistema armazena as informações do novo usuário.                                                                                                                                                                                 |
| **Garantias mínimas**         | Nenhum cadastro incompleto é concluído indevidamente. <br> Os dados sensíveis são tratados com segurança.                                                                                                                                                                                            |
| Garantias de sucesso          | O usuário é cadastrado com sucesso e pode acessar o sistema. <br> O sistema armazena as informações do novo usuário.                                                                                                                                                                                 |
| **Fluxo principal**           | 1. O usuário acessa a tela de cadastro. <br> 2. O sistema solicita os dados cadastrais necessários. <br> 3. O usuário informa os dados obrigatórios. <br> 4. O sistema valida as informações fornecidas. <br> 5. O sistema cria a conta do usuário. <br> 6. O sistema confirma o cadastro realizado. |
| **Extensões  **               | 4a. Dados inválidos ou inconsistentes: o sistema informa o erro e solicita correção. <br> 4b. Usuário já cadastrado: o sistema informa que já existe conta associada. <br> 5a. O sistema exibe mensagem de erro e solicita correção.                                                                 |
| **Regras de negócio**         | RN-01: O sistema deve validar que o email fornecido é único. <br> RN-02: O sistema deve garantir que a senha atenda aos critérios de segurança estabelecidos. <br> RN-03: O sistema deve armazenar as informações do usuário de forma segura, utilizando criptografia para dados sensíveis.          |
| **Requisitos especiais**      | O sistema deve armazenar senha de forma segura <br> O cadastro deve funcionar em dispositivos móveis e desktop.                                                                                                                                                                                      |
