# Cockburn dos requisitos

## Caso de uso de cadastro de usuário

| Campo                         | Descrição                                                                                                                                                                                                                                                                                            |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                        | UC-01                                                                                                                                                                                                                                                                                                |
| **Nome**                      | Cadastrar usuário                                                                                                                                                                                                                                                                                    |
| **Objetivo**                  | Permitir que um novo usuário se cadastre no sistema                                                                                                                                                                                                                                                  |
| **Ator primário**             | Usuário                                                                                                                                                                                                                                                                                              |
| **Interessados e Interesses** | **Usuário:** deseja criar conta para acessar o sistema. <br> **UESPI/Ouvidoria:** deseja permitir acesso apenas a usuários devidamente cadastrados.                                                                                                                                                  |
| **Gatilho**                   | O usuário seleciona a opção de cadastro                                                                                                                                                                                                                                                              |
| **Pré-condições**             | O usuário ainda não possui cadastro ativo no sistema.                                                                                                                                                                                                                                                |
| **Pós-condições**             | O usuário é cadastrado com sucesso e pode acessar o sistema. <br> O sistema armazena as informações do novo usuário.                                                                                                                                                                                 |
| **Garantias mínimas**         | Nenhum cadastro incompleto é concluído indevidamente. <br> Os dados sensíveis são tratados com segurança.                                                                                                                                                                                            |
| **Garantias de sucesso**      | O usuário é cadastrado com sucesso e pode acessar o sistema. <br> O sistema armazena as informações do novo usuário.                                                                                                                                                                                 |
| **Fluxo principal**           | 1. O usuário acessa a tela de cadastro. <br> 2. O sistema solicita os dados cadastrais necessários. <br> 3. O usuário informa os dados obrigatórios. <br> 4. O sistema valida as informações fornecidas. <br> 5. O sistema cria a conta do usuário. <br> 6. O sistema confirma o cadastro realizado. |
| **Extensões**                 | 4.1 Dados inválidos ou inconsistentes: o sistema informa o erro e solicita correção. <br> 4.2 Usuário já cadastrado: o sistema informa que já existe conta associada. <br> 5.1 Erro ao concluir cadastro: o sistema exibe mensagem de erro e solicita nova tentativa.                                |
| **Regras de negócio**         | RN-01: O sistema deve validar que o e-mail fornecido é único. <br> RN-02: O sistema deve garantir que a senha atenda aos critérios de segurança estabelecidos. <br> RN-03: O sistema deve armazenar as informações do usuário de forma segura, utilizando criptografia para dados sensíveis.         |
| **Requisitos especiais**      | O sistema deve armazenar senha de forma segura. <br> O cadastro deve funcionar em dispositivos móveis e desktop.                                                                                                                                                                                     |

# Caso de uso de autenticação

| Campo                         | Descrição                                                                                                                                                                                                                                                              |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                        | UC-02                                                                                                                                                                                                                                                                  |
| **Nome**                      | Autenticar usuário                                                                                                                                                                                                                                                     |
| **Objetivo**                  | Permitir que um usuário autenticado acesse o sistema com segurança                                                                                                                                                                                                     |
| **Ator primário**             | Usuário                                                                                                                                                                                                                                                                |
| **Interessados e Interesses** | **Usuário:** deseja acessar sua conta para utilizar o sistema. <br> **UESPI/Ouvidoria:** deseja garantir acesso somente a perfis autorizados.                                                                                                                          |
| **Gatilho**                   | O usuário seleciona a opção de login                                                                                                                                                                                                                                   |
| **Pré-condições**             | O usuário já possui cadastro ativo no sistema.                                                                                                                                                                                                                         |
| **Pós-condições**             | O usuário inicia sessão no sistema. <br> O sistema identifica o perfil de acesso do usuário.                                                                                                                                                                           |
| **Garantias mínimas**         | Tentativas inválidas não concedem acesso indevido. <br> As credenciais são processadas de forma segura.                                                                                                                                                                |
| **Garantias de sucesso**      | O usuário é autenticado e redirecionado para a área correspondente ao seu perfil.                                                                                                                                                                                      |
| **Fluxo principal**           | 1. O usuário acessa a tela de autenticação. <br> 2. O sistema solicita e-mail e senha. <br> 3. O usuário informa as credenciais. <br> 4. O sistema valida as credenciais. <br> 5. O sistema autentica o usuário. <br> 6. O sistema libera o acesso à área autenticada. |
| **Extensões**                 | 4.1 Credenciais inválidas: o sistema informa erro e solicita nova tentativa. <br> 4.2 Usuário esqueceu a senha: o sistema direciona para recuperação de senha.                                                                                                         |
| **Regras de negócio**         | RN-01: Apenas usuários cadastrados podem se autenticar. <br> RN-02: O acesso deve respeitar o perfil do usuário. <br> RN-03: O sistema deve registrar eventos de autenticação.                                                                                         |
| **Requisitos especiais**      | O processo de login deve ser responsivo. <br> O tempo de resposta da autenticação deve respeitar os limites de desempenho definidos.                                                                                                                                   |

## Caso de uso de recuperação de senha

| Campo                         | Descrição                                                                                                                                                                                                                                                                                                         |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                        | UC-03                                                                                                                                                                                                                                                                                                             |
| **Nome**                      | Recuperar senha                                                                                                                                                                                                                                                                                                   |
| **Objetivo**                  | Permitir que o usuário redefina a senha de acesso quando não conseguir se autenticar                                                                                                                                                                                                                              |
| **Ator primário**             | Usuário                                                                                                                                                                                                                                                                                                           |
| **Interessados e Interesses** | **Usuário:** deseja voltar a acessar sua conta. <br> **UESPI/Ouvidoria:** deseja oferecer recuperação segura sem comprometer a conta.                                                                                                                                                                             |
| **Gatilho**                   | O usuário seleciona a opção de recuperação de senha                                                                                                                                                                                                                                                               |
| **Pré-condições**             | O usuário possui cadastro no sistema e acesso ao canal de recuperação definido.                                                                                                                                                                                                                                   |
| **Pós-condições**             | A senha do usuário é redefinida com sucesso. <br> O sistema registra a operação de recuperação.                                                                                                                                                                                                                   |
| **Garantias mínimas**         | Nenhuma senha é alterada sem validação adequada do usuário. <br> Códigos inválidos ou expirados não são aceitos.                                                                                                                                                                                                  |
| **Garantias de sucesso**      | O usuário redefine a senha e pode voltar a se autenticar normalmente.                                                                                                                                                                                                                                             |
| **Fluxo principal**           | 1. O usuário acessa a opção de recuperação de senha. <br> 2. O sistema solicita o identificador da conta. <br> 3. O usuário informa o e-mail cadastrado. <br> 4. O sistema envia o código ou link de recuperação. <br> 5. O usuário informa o código e a nova senha. <br> 6. O sistema valida e redefine a senha. |
| **Extensões**                 | 3.1 Usuário não encontrado: o sistema informa que não localizou a conta. <br> 5.1 Código inválido: o sistema informa erro e solicita novo código. <br> 5.2 Código expirado: o sistema exige nova solicitação de recuperação.                                                                                      |
| **Regras de negócio**         | RN-01: A recuperação deve ocorrer por canal previamente associado à conta. <br> RN-02: O código de recuperação deve possuir validade limitada. <br> RN-03: A nova senha deve seguir os critérios de segurança do sistema.                                                                                         |
| **Requisitos especiais**      | O sistema deve enviar a recuperação por meio seguro. <br> A redefinição de senha deve ser compatível com dispositivos móveis e desktop.                                                                                                                                                                           |

## caso de uso de registro de manifestação

| Campo                         | Descrição                                                                                                                                                                                                                                                                                                                                                                                  |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **ID**                        | UC-04                                                                                                                                                                                                                                                                                                                                                                                      |
| **Nome**                      | Registrar manifestação                                                                                                                                                                                                                                                                                                                                                                     |
| **Objetivo**                  | Permitir que o usuário registre uma reclamação, denúncia, solicitação, sugestão ou elogio                                                                                                                                                                                                                                                                                                  |
| **Ator primário**             | Usuário                                                                                                                                                                                                                                                                                                                                                                                    |
| **Interessados e Interesses** | **Usuário:** deseja formalizar uma manifestação e receber protocolo. <br> **UESPI/Ouvidoria:** deseja registrar e organizar manifestações para tratamento adequado.                                                                                                                                                                                                                        |
| **Gatilho**                   | O usuário seleciona a opção de registrar manifestação                                                                                                                                                                                                                                                                                                                                      |
| **Pré-condições**             | O usuário está autenticado, exceto nos casos permitidos de denúncia anônima ou sigilosa conforme regra institucional.                                                                                                                                                                                                                                                                      |
| **Pós-condições**             | A manifestação é registrada com sucesso. <br> O sistema gera um número de protocolo único.                                                                                                                                                                                                                                                                                                 |
| **Garantias mínimas**         | O sistema não registra manifestação sem os dados obrigatórios. <br> Nenhum protocolo é gerado para envio incompleto.                                                                                                                                                                                                                                                                       |
| **Garantias de sucesso**      | A manifestação é salva, vinculada a campus e unidade administrativa, e fica disponível para acompanhamento.                                                                                                                                                                                                                                                                                |
| **Fluxo principal**           | 1. O usuário acessa a funcionalidade de registro. <br> 2. O sistema apresenta o formulário de manifestação. <br> 3. O usuário seleciona tipo, campus e unidade administrativa. <br> 4. O usuário preenche a descrição e demais informações. <br> 5. O usuário, se desejar, inclui anexos. <br> 6. O sistema valida os dados. <br> 7. O sistema registra a manifestação e gera o protocolo. |
| **Extensões**                 | 4.1 Dados ausentes: o sistema informa os campos obrigatórios não preenchidos. <br> 5.1 Necessidade de anexos: o sistema inclui a funcionalidade de anexar arquivos à manifestação. <br> 6.1 Dados inconsistentes: o sistema impede o registro até a correção.                                                                                                                              |
| **Regras de negócio**         | RN-01: Toda manifestação deve possuir protocolo único. <br> RN-02: Toda manifestação deve estar vinculada a um campus e a uma unidade administrativa. <br> RN-03: O usuário deve informar o tipo de manifestação. <br> RN-04: O sistema pode permitir sigilo ou anonimato conforme política institucional.                                                                                 |
| **Requisitos especiais**      | O formulário deve ser acessível e responsivo. <br> O sistema deve registrar a manifestação em até tempo aceitável de uso.                                                                                                                                                                                                                                                                  |

| **Pós-condições**             | O arquivo é anexado com sucesso à manifestação.                                                                                                                                                                                                                                                 |

O sistema valida formato e tamanho. <br> 5. O sistema armazena o arquivo e o associa à manifestação. <br> 6. O sistema confirma o anexo realizado. |

| **Regras de negócio**         | RN-01: O sistema deve validar tamanho e formato dos anexos. <br> RN-02: Chamados encerrados não podem receber novos anexos pelo usuário. <br> RN-03: O sistema deve bloquear arquivos potencialmente maliciosos.                                                                                |
| **Requisitos especiais**      | O envio deve ocorrer com segurança e validação antivírus, quando aplicável. <br> O sistema deve preservar integridade e disponibilidade dos anexos.                                                                                                                                             |

| **Requisitos especiais**      | A consulta deve ter resposta rápida e interface compatível com desktop e mobile.                                                                                                                                                  |

| **Ator primário**             | Usuário                                                                                                                                                                                                                             |

| **Requisitos especiais**      | O sistema deve preservar histórico completo da comunicação. <br> O envio deve disparar notificações quando aplicável.                                                                                                               |

## Caso de uso de encerramento de manifestação

| Campo                         | Descrição                                                                                                                                                                                                                                            |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                        | UC-08                                                                                                                                                                                                                                                |
| **Nome**                      | Encerrar manifestação                                                                                                                                                                                                                                |
| **Objetivo**                  | Permitir que o usuário encerre a manifestação quando considerar a demanda resolvida                                                                                                                                                                  |
| **Ator primário**             | Usuário                                                                                                                                                                                                                                              |
| **Interessados e Interesses** | **Usuário:** deseja encerrar o chamado após solução satisfatória. <br> **UESPI/Ouvidoria:** deseja manter o ciclo de atendimento corretamente finalizado.                                                                                            |
| **Gatilho**                   | O usuário seleciona a opção de encerrar o chamado                                                                                                                                                                                                    |
| **Pré-condições**             | A manifestação existe, pertence ao usuário e possui condições para encerramento.                                                                                                                                                                     |
| **Pós-condições**             | O chamado é marcado como encerrado. <br> O histórico registra a ação de fechamento.                                                                                                                                                                  |
| **Garantias mínimas**         | O sistema não encerra demanda sem registrar data e responsável pela ação.                                                                                                                                                                            |
| **Garantias de sucesso**      | O chamado passa ao estado encerrado e fica disponível apenas para consulta e avaliação, conforme regra de negócio.                                                                                                                                   |

| **Extensões**                 | 2.1 Manifestação sem resposta conclusiva: o sistema bloqueia o encerramento, conforme regra do processo. <br> 3.1 Usuário desiste da ação: o sistema mantém o chamado em aberto.                                                                     |
| **Regras de negócio**         | RN-01: Um chamado só pode ser encerrado após resposta da equipe ou por solicitação válida do usuário. <br> RN-02: Chamados encerrados não podem ser editados pelo usuário. <br> RN-03: O fechamento deve ser registrado em histórico.                |
| **Requisitos especiais**      | O sistema deve exibir confirmação clara antes do encerramento.                                                                                                                                                                                       |

| **Objetivo**                  | Direcionar o usuário para atendimento humano quando o chatbot não puder responder com segurança                                                                                                                                                                 |
| **Ator primário**             | Usuário                                                                                                                                                                                                                                                         |
| **Interessados e Interesses** | **Usuário:** deseja receber suporte adequado em situações complexas. <br> **UESPI/Ouvidoria:** deseja evitar respostas imprecisas e manter qualidade no atendimento.                                                                                            |
| **Gatilho**                   | O chatbot identifica situação complexa, sensível, fora de escopo ou de baixa confiança                                                                                                                                                                          |
| **Pré-condições**             | Existe interação ativa entre usuário e chatbot.                                                                                                                                                                                                                 |
| **Pós-condições**             | O usuário é orientado a abrir manifestação ou seguir para atendimento humano.                                                                                                                                                                                   |
| **Garantias mínimas**         | O sistema não mantém o usuário preso a uma resposta automatizada inadequada.                                                                                                                                                                                    |

| **Fluxo principal**           | 1. O chatbot identifica que não pode responder com segurança. <br> 2. O sistema informa a limitação ao usuário. <br> 3. O sistema apresenta a opção de encaminhamento para atendimento humano ou abertura de chamado. <br> 4. O usuário segue o canal indicado. |
| **Extensões**                 | 3.1 Usuário não deseja prosseguir: o sistema encerra a interação mantendo o registro da conversa.                                                                                                                                                               |
| **Regras de negócio**         | RN-01: O encaminhamento deve ocorrer em casos de baixa confiança, pergunta sensível ou assunto fora de escopo. <br> RN-02: O chatbot deve orientar o usuário sem tomar decisão administrativa.                                                                  |
| **Requisitos especiais**      | A transição para atendimento humano deve ser clara e rastreável.                                                                                                                                                                                                |

## caso de uso de gerenciamento de manifestações

| Campo                         | Descrição                                                                                                                                                                                                                                                                                                                          |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                        | UC-12                                                                                                                                                                                                                                                                                                                              |
| **Nome**                      | Gerenciar manifestações                                                                                                                                                                                                                                                                                                            |
| **Objetivo**                  | Permitir que a equipe da ouvidoria visualize, filtre, atribua e atualize os chamados registrados                                                                                                                                                                                                                                   |
| **Ator primário**             | Ouvidor                                                                                                                                                                                                                                                                                                                            |
| **Interessados e Interesses** | **Ouvidor:** deseja tratar manifestações com eficiência e rastreabilidade. <br> **UESPI/Ouvidoria:** deseja garantir resposta adequada e controle do fluxo de atendimento.                                                                                                                                                         |
| **Gatilho**                   | O ouvidor acessa a área administrativa de gerenciamento                                                                                                                                                                                                                                                                            |
| **Pré-condições**             | O ouvidor está autenticado e possui permissão administrativa adequada.                                                                                                                                                                                                                                                             |
| **Pós-condições**             | As manifestações consultadas ou alteradas permanecem com histórico atualizado.                                                                                                                                                                                                                                                     |
| **Garantias mínimas**         | Usuários sem permissão não acessam nem alteram chamados administrativos.                                                                                                                                                                                                                                                           |
| **Garantias de sucesso**      | O ouvidor consegue localizar, atribuir, comentar, atualizar status e acompanhar os chamados sob sua responsabilidade.                                                                                                                                                                                                              |
| **Fluxo principal**           | 1. O ouvidor acessa a área de gerenciamento. <br> 2. O sistema lista os chamados registrados. <br> 3. O ouvidor aplica filtros por status, data, campus, unidade ou gravidade. <br> 4. O ouvidor seleciona um chamado. <br> 5. O sistema exibe detalhes e histórico. <br> 6. O ouvidor atualiza status, atribuição ou comentários. |
| **Extensões**                 | 3.1 Nenhum resultado encontrado: o sistema informa a ausência de chamados compatíveis com o filtro. <br> 6.1 Alteração não autorizada: o sistema bloqueia a operação.                                                                                                                                                              |
| **Regras de negócio**         | RN-01: Apenas atendentes e administradores autorizados podem alterar status e atribuição. <br> RN-02: O sistema deve manter histórico completo das ações administrativas. <br> RN-03: Chamados sigilosos devem ter acesso restrito a perfis autorizados.                                                                           |
| **Requisitos especiais**      | A tela administrativa deve suportar filtros e atualização com desempenho adequado.                                                                                                                                                                                                                                                 |

## Caso de uso de geração de relatórios gerenciais

| Campo                         | Descrição                                                                                                                                                                                                                                                                                       |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                        | UC-13                                                                                                                                                                                                                                                                                           |
| **Nome**                      | Gerar relatórios                                                                                                                                                                                                                                                                                |
| **Objetivo**                  | Permitir a geração de relatórios gerenciais sobre as manifestações registradas                                                                                                                                                                                                                  |
| **Ator primário**             | Administrador                                                                                                                                                                                                                                                                                   |
| **Interessados e Interesses** | **Administrador:** deseja obter indicadores para análise e tomada de decisão. <br> **UESPI/Ouvidoria:** deseja acompanhar volume, tempo de resposta e satisfação dos usuários.                                                                                                                  |
| **Gatilho**                   | O administrador seleciona a opção de gerar relatórios                                                                                                                                                                                                                                           |
| **Pré-condições**             | O administrador está autenticado e possui acesso aos dados gerenciais.                                                                                                                                                                                                                          |
| **Pós-condições**             | O relatório é gerado conforme filtros e parâmetros definidos.                                                                                                                                                                                                                                   |
| **Garantias mínimas**         | O sistema não expõe informações além do perfil autorizado.                                                                                                                                                                                                                                      |
| **Garantias de sucesso**      | O administrador obtém relatório consolidado com estatísticas e indicadores relevantes.                                                                                                                                                                                                          |
| **Fluxo principal**           | 1. O administrador acessa a área de relatórios. <br> 2. O sistema apresenta filtros por período, tipo, campus, unidade e status. <br> 3. O administrador define os parâmetros desejados. <br> 4. O sistema processa os dados. <br> 5. O sistema gera o relatório com estatísticas e tendências. |
| **Extensões**                 | 3.1 Parâmetros inválidos: o sistema solicita ajuste dos filtros. <br> 4.1 Ausência de dados para o período: o sistema informa que não há registros suficientes para compor o relatório.                                                                                                         |
| **Regras de negócio**         | RN-01: Os relatórios devem considerar indicadores como tempo médio de resposta, tempo de resolução, volume por categoria e índice de satisfação. <br> RN-02: Somente usuários autorizados podem gerar relatórios gerenciais.                                                                    |
| **Requisitos especiais**      | O processamento deve ocorrer em tempo aceitável. <br> Os relatórios devem preservar privacidade de dados pessoais quando aplicável.                                                                                                                                                             |
