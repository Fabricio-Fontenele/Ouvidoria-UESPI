# ouvidoria UESPI

#Requisitos Funcionais
- O sistema deve permitir que os usuários se cadastrem utilizando um endereço de e-mail válido.
- O sistema deve permitir que os usuários façam login utilizando seu endereço de e-mail e senha
- O sistema deve permitir que os usuários recuperem suas senhas caso as esqueçam.
- O sistema deve permitir que os usuários atualizem suas informações pessoais, como nome e endereço.
- O sistema deve permitir que os usuários registrem suas reclamações, denúncias, solicitações, sugestões, fornecendo detalhes e anexando arquivos, se necessário.
- O sistema deve permitir que os usuários visualizem o status de suas reclamações, denúncias, solicitações, sugestões incluindo atualizações e respostas da equipe de atendimento.
- O sistema deve permitir que os usuários recebam notificações por e-mail sobre o status de suas reclamações, denúncias, solicitações, sugestões ou elogios.
- O sistema deve permitir que os administradores visualizem e gerenciem as reclamações, denúncias, solicitações, sugestões registrados pelos usuários, incluindo a capacidade de responder e atualizar o status.
- O sistema deve permitir que os administradores gerem relatórios sobre as reclamações, denúncias, solicitações, sugestões ou elogios registrados, incluindo estatísticas e tendências.
- O sistema deve permitir que os usuários possam tirar duvidas e obter informações sobre a instituição, editais, cursos, etc, por meio de um chatBot(GuaraPI) integrado ao sistema, proporcionando um atendimento rápido e eficiente.

#Requisitos não funcionais
- O sistema deve ser acessível por meio de dispositivos móveis e desktops, garantindo uma experiência de usuário consistente em todas as plataformas.
- O sistema deve garantir a segurança dos dados dos usuários, implementando medidas de proteção contra acesso não autorizado e vazamento de informações.
